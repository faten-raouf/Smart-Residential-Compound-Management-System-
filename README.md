# Smart Residential Compound Management System

A comprehensive management system designed to simulate daily operations inside a smart residential compound. The system supports multiple user roles with specific permissions to ensure efficient management of compound services.

## 🛠️ Built With
- *Language:* C++[span_1](start_span)[span_1](end_span)
- *Concepts Used:* Object-Oriented Programming (OOP)[span_2](start_span)[span_2](end_span), Data Structures, and Database Design concepts.

## 👥 User Roles & Features

### 1. Resident
- Register a new account[span_3](start_span)[span_3](end_span).
- Create maintenance and service requests (e.g., Plumbing, Electricity, Carpentry)[span_4](start_span)[span_4](end_span).
- File complaints with details and dates[span_5](start_span)[span_5](end_span).
- Book compound facilities (e.g., Gym Sessions, Pool Slots, Event Halls)[span_6](start_span)[span_6](end_span).

### 2. Admin
- Manage and save system configuration data[span_7](start_span)[span_7](end_span).
- Access and monitor all compound data and services[span_8](start_span)[span_8](end_span).

### 3. Technician
- Receive and manage assigned service requests[span_9](start_span)[span_9](end_span).
- Update request status (Pending, In Progress, Completed)[span_10](start_span)[span_10](end_span).
